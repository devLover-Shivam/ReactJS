# React Overview

## Introduction

ReactJS is a powerful JavaScript library developed by Meta (formerly Facebook) in 2013. It is primarily used for building dynamic, interactive, and scalable user interfaces (UI) for web applications.

---

## Why React Was Created

Around 2010–2011, platforms such as Facebook experienced rapid user growth. Traditional web development using HTML, CSS, and JavaScript faced challenges when updating user interfaces. Even small changes, such as updating notification counts or message counters, often required reloading large portions of a webpage.

This approach became inefficient as applications grew in complexity and scale.

---

## Component-Based Architecture

React introduced a **Component-Based Architecture**, where a user interface is divided into small, reusable, and independent components.

### Benefits

* Improves code reusability and maintainability.
* Allows developers to update specific sections of a webpage without refreshing the entire page.
* Enhances application performance and user experience.
* Makes large-scale applications easier to manage and develop.

For example, components such as navigation bars, notification panels, chat windows, and user profiles can be updated independently without affecting the rest of the application.

---

## Career Relevance

React is one of the most widely adopted frontend technologies in the industry and continues to be among the most in-demand skills for frontend developers.

Knowledge of React is valuable for:

* Frontend Development
* Full-Stack Development
* Modern Web Application Development
* Single Page Applications (SPAs)

---

## Prerequisites for Learning React

Before learning React, developers should have a strong understanding of the following technologies:

### HTML

* Semantic elements
* Forms
* Document structure

### CSS

* Selectors
* Flexbox
* Grid
* Responsive Design

### JavaScript

* Variables and Data Types
* Functions
* Objects and Arrays
* ES6+ Features
* DOM Manipulation
* Asynchronous Programming

A solid foundation in HTML, CSS, and JavaScript makes it significantly easier to understand React concepts and development practices.


# Library vs Framework

Before learning React, it's important to understand the difference between a **Library** and a **Framework**.

---

## What is a Library?

A **Library** is a collection of pre-written code that helps developers perform specific tasks. Developers decide when and how to use it.

### Examples

### ReactJS

**Purpose:** Building User Interfaces (UI)

React helps create reusable UI components and update only the parts of a webpage that change.

**Use Cases:**

* Social Media Apps
* Dashboards
* E-commerce Websites

---

### GSAP

**Purpose:** Creating Animations

GSAP is used to build smooth and high-performance animations.

**Use Cases:**

* Landing Pages
* Scroll Animations
* Interactive Websites

---

### Lenis

**Purpose:** Smooth Scrolling

Lenis provides a modern and fluid scrolling experience.

**Use Cases:**

* Portfolio Websites
* Agency Websites
* Creative Projects

---

## What is a Framework?

A **Framework** provides a complete structure for building applications. It comes with predefined rules, architecture, and tools.

> With a library, you call the code.
> With a framework, the framework calls your code.

### Examples

### Next.js

**Purpose:** Full-Stack React Framework

Next.js extends React by providing features such as routing, server-side rendering, API routes, and SEO optimization.

**Use Cases:**

* Blogs
* SaaS Applications
* E-commerce Platforms

---

### Angular

**Purpose:** Enterprise Web Applications

Angular is a complete frontend framework developed by Google.

**Use Cases:**

* Banking Applications
* Enterprise Dashboards
* Large-Scale Business Systems
